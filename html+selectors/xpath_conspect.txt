НАКОНЕЦ-ТО XPATH!!!
1. Абсолютный путь (absolute path):
абсолютный путь селектора xpath c указанием индкса[n] элемента по абсолютному пути в контейнере div:
    html/body/div/p[4]
2. Контекстный селектор:
пишется через два "//" - чтобы найти все элементы данного тега в файле и может также использовать индекс [n], 
а также по любому атрибуту
    //p[3] - Контекстный по индексу
    //p[@id="id2"] - Контекстный по атрибуту
    //p[@class="неуникальный_класс_элемента"] - Контекстный по атрибуту- класс
3. Текстовый селектор
    //li[text()="ненумерованный список: строка 3"] - Текстовый по полному совпадению текста
    //li[contains(text(), "строка 4")] - Текстовый по частичному совпадению текста
4. Частичный по атрибуту
    //div[contains(@class, "for")] - по частичному совпадению атрибута
    //h2[contains(@id, "id")] - тоже самое для примера разных атрибутов
    //*[contains(@id, "id")] - все элементы с параметром id, содержащим "id". * - как и везде - "всё"
5. Начинаестя с...
    //*[starts-with(@class, "неун")] - все элементы, класс которых начинается с "неун"
6. Составные (сложные) запросы
    //div[@class="different_elements"]//ul/li[5] - в контейнере с классом different_elements из ненумерованного списка взять элемент с индексом [5]
    //h2[text()="при помощи pytest & playwright" or text()="7"] - для условий можно использовать логические операторы: and, or
    //h2[text()="при помощи pytest & playwright" and @class="неуникальный_класс_элемента class2"] - and
